# RAG视频知识库检索

## 一、项目背景
日常使用各种视频平台的过程中，都会遇到过一些内容，可能是感兴趣的，可能是视频中提到的内容很有用，可能是涉及到的知识很详细，等等，这些视频因为当前的种种原因（可能是因为当前用不上、没时间等原因）没办法当场掌握，而把它收藏到收藏夹中，但是在后续又会因为收藏内容过多、很难找到具体的内容而导致这些视频在收藏夹中吃灰。因此，本项目主要是想做的内容是将收藏夹中的视频内容提取出来作为一个知识库的存在，最后再通过RAG去进行检索，这样就能够解决好的内容收藏夹吃灰的问题。

## 二、初步设想

### 2.1 项目环境
主要平台：Dify

### 2.2 项目技术
- 工作流
- RAG检索增强生成

### 2.3 主要内容
在Dify中创建一个工作流，作用是从主流平台从获取具体的视频内容，再将获取到的内容进行一定的处理，把处理后的内容发送对应的大模型进行解析、总结，最后生成一个关于当前视频的说明文档 ，文档中的内容包括视频的文案，关键内容整理以及整个视频内容的总结。生成的文件会被保存到一个特定的位置，这个文件夹会作为RAG检索的目标知识库。用户可以在聊天框中与LLM进行交互，从而要求其在知识库中检索到相关内容， 再总结返回给用户。

### 2.4 主要流程
1. 自动从目标位置中获取视频内容；
2. 对视频进行处理（包括抽帧等内容）；
3. 大模型将处理后内容进行解析、整理；
4. 将整理后的内容作为一个新的文件内容保存到知识库中；
5. 用户进行交互；
6. LLM在知识库中进行RAG检索；
7. 将检索到的内容总结返回给用户。
